# Zotero 科研通文献求助助手设计

## 1. 目标

为 Zotero `9.0-beta.21+1a89239a1`（64 位）开发一套本地插件组合。用户在 Zotero 中选择一篇已有元数据但没有 PDF 的文献后，可以一键打开科研通文献求助页面，自动填入关键信息，并在人工核对后手动发布求助。

首版同时支持 Chrome 和 Edge。系统不得自动点击科研通最终发布按钮，不保存科研通账号、Cookie 或密码，也不绕过登录、验证码、积分确认或网站的其他风控机制。

## 2. 交付物

项目交付两个可安装组件及中文说明：

1. Zotero 插件安装包 `.xpi`。
2. Chrome/Edge 共用的 Manifest V3 浏览器扩展目录及压缩包。
3. 中文安装、配置、使用和故障排查文档。
4. 自动化测试、构建脚本与版本兼容性说明。

## 3. 范围

### 3.1 首版包含

- 只处理当前选中的一篇普通 Zotero 文献条目。
- Zotero 工具栏按钮“科研通求助”。
- 文献列表右键菜单“在科研通发起文献求助”。
- 提取 DOI、标题、作者、期刊或出版物、年份、URL 等元数据。
- 检查子附件中是否存在 PDF。
- 打开 `https://www.ablesci.com/assist/create`。
- 将 Zotero 元数据通过 URL 片段交给浏览器扩展。
- 科研通未登录时保留当前任务，登录后继续填表。
- 有 DOI 时自动填写 DOI 并触发科研通查询。
- 用 Zotero 元数据补齐科研通未返回的空字段。
- 对两边不一致的字段显示差异提示，但保留科研通查询结果。
- 自动操作失败时提供完整元数据复制功能。

### 3.2 首版不包含

- 多选、批量或连续发布求助。
- 自动点击最终发布按钮。
- 自动处理验证码、积分、悬赏或高级求助选项。
- 读取、导出或复用浏览器 Cookie、密码和会话凭据。
- 直接调用科研通未公开接口提交求助。
- Firefox、Safari 或其他非 Chromium 浏览器扩展。
- 自动抓取或下载 PDF。

## 4. 总体架构

系统由 Zotero 插件和 Chromium 浏览器扩展组成，不引入常驻本机服务。

### 4.1 Zotero 插件

Zotero 插件采用 `manifest.json` 与 `bootstrap.js` 启动结构，并将兼容范围限定到实际验证过的 Zotero 9 小版本。插件负责：

- 注册工具栏按钮和文献列表右键菜单。
- 获取当前活动 Zotero 窗口中的选中条目。
- 执行条目类型、选择数量、标题和 PDF 附件检查。
- 标准化并序列化文献元数据。
- 生成带临时 URL 片段的科研通地址。
- 将可读版元数据写入剪贴板作为降级通道。
- 使用系统默认浏览器打开科研通。
- 显示中文成功或错误提示。

### 4.2 浏览器扩展

浏览器扩展使用 Manifest V3，仅申请 `ablesci.com` 所需的最小站点权限。扩展负责：

- 在科研通登录页和求助创建页运行内容脚本。
- 解析、校验 URL 片段中的临时载荷。
- 解析成功后立即清除地址栏中的载荷。
- 将待处理任务暂存在 `chrome.storage.session`。
- 登录跳转后恢复任务并导航到求助创建页。
- 驱动 DOI 查询及字段补齐。
- 显示状态条、差异信息和手动复制按钮。
- 在页面结构不匹配时停止自动操作，不执行猜测性点击。

### 4.3 通信边界

Zotero 与浏览器扩展之间不使用网络服务、Native Messaging 或剪贴板读取。主通信通道为 URL `#` 片段，片段不会作为 HTTP 请求的一部分发送给科研通服务器。

剪贴板仅由 Zotero 写入一份便于人工粘贴的元数据文本。浏览器扩展不依赖剪贴板权限。

## 5. 数据模型

临时载荷采用带版本号的 JSON，并进行 Base64URL 编码：

```json
{
  "v": 1,
  "createdAt": 1781530000000,
  "nonce": "random-id",
  "item": {
    "zoteroKey": "ABCD1234",
    "itemType": "journalArticle",
    "title": "Article title",
    "doi": "10.1000/example",
    "authors": ["Family, Given"],
    "publicationTitle": "Journal title",
    "year": "2026",
    "url": "https://publisher.example/article"
  }
}
```

约束如下：

- `v` 用于载荷格式迁移。
- `createdAt` 用于拒绝过期任务，默认有效期 30 分钟。
- `nonce` 用于识别同一浏览器会话内重复载荷。
- `title` 必填；其他字段可为空。
- 字符串在编码前去除首尾空白并限制最大长度。
- URL 只接受 `http:` 或 `https:`。
- DOI 去除 `https://doi.org/`、`http://dx.doi.org/` 和 `doi:` 前缀后保存。

## 6. Zotero 操作流程

1. 用户选择一条文献并点击工具栏按钮，或从右键菜单启动。
2. 插件确认恰好选择了一条记录。
3. 插件拒绝附件、笔记、批注和集合等非普通文献条目。
4. 插件确认标题不为空，否则停止并提示补充标题。
5. 插件读取所有子附件。若任一附件的内容类型为 `application/pdf`，或明确表示为本地、链接或远程 PDF，则停止并提示“该文献已有 PDF，无需发起求助”。
6. 插件提取并标准化元数据。
7. 插件将可读版元数据写入剪贴板。
8. 插件生成科研通 URL 并交给系统默认浏览器打开。
9. 插件提示“已打开科研通，请核对后发布”。

如果默认浏览器没有安装配套扩展，科研通页面仍会打开，用户可以使用剪贴板内容手动填写。

## 7. 科研通填表流程

### 7.1 载荷接收

内容脚本仅识别约定前缀的 URL 片段。解析、版本、字段类型或有效期检查失败时，不操作页面，并显示可理解的中文错误。

载荷验证成功后：

1. 写入浏览器会话存储。
2. 使用 `history.replaceState()` 清除地址栏片段。
3. 状态条显示“正在读取 Zotero 元数据”。

### 7.2 登录恢复

如果当前页面是登录页或页面显示未登录状态：

- 保留会话任务，不填写账号信息。
- 显示“请先登录科研通，登录后将继续自动填写”。
- 登录成功后导航或恢复到求助创建页。
- 浏览器会话关闭、任务过期或用户取消后删除任务。

### 7.3 DOI 优先流程

有 DOI 时：

1. 定位 DOI 输入框并填写标准化 DOI。
2. 定位明确的“查询”操作并自动点击。
3. 等待科研通返回元数据或显示失败状态。
4. 查询成功时，科研通结果为主。
5. Zotero 数据只补齐空字段。
6. 标题、DOI、官网链接等非空字段不一致时，显示黄色差异提示，不自动覆盖。

### 7.4 无 DOI 或查询失败

没有 DOI，或 DOI 查询超时、明确失败时：

- 进入手动字段填充流程。
- 填入标题、作者、期刊、年份和 URL 等可识别字段。
- 没有 DOI 时提示用户重点核对并补充官网链接。
- 不把第三方索引链接伪装为出版社官网链接。

### 7.5 最终状态

填表完成后显示“已自动填充，请核对后发布”。扩展不得定位、触发或模拟最终发布按钮。最终提交必须由用户主动完成。

## 8. 页面适配策略

科研通没有公开稳定的填表 API，因此页面适配采用可维护的选择器策略：

- 优先使用字段标签、`name`、稳定属性和按钮可见文本。
- 每类字段允许一组按优先级排列的候选选择器。
- 页面操作前检查元素唯一性、可见性和可用性。
- DOI 查询采用明确的状态变化、字段变化或错误提示作为完成条件，不仅依赖固定延迟。
- 任一关键控件无法唯一识别时立即停止自动操作。
- 选择器映射与业务逻辑分离，便于网站改版后小范围更新。

开发期需要使用测试账号人工检查登录后的真实求助页面 DOM。测试账号、密码和 Cookie 不进入源码、测试夹具或日志。

## 9. 用户界面

### 9.1 Zotero

- 工具栏按钮：`科研通求助`。
- 右键菜单：`在科研通发起文献求助`。
- 两个入口调用同一命令处理器。
- 中文提示覆盖未选、多选、错误条目类型、标题缺失、已有 PDF 和打开失败。

### 9.2 科研通页面状态条

状态条固定在页面边缘，不遮挡主要表单，包含：

- 当前阶段文本。
- 成功、警告和错误三种视觉状态。
- 字段差异列表。
- `复制全部元数据` 按钮。
- `取消本次填充` 按钮。

状态条不包含“发布”或任何等效提交操作。

## 10. 安全与隐私

- 扩展站点权限限制为科研通域名。
- URL 载荷不包含账号、密码、Cookie、笔记正文或 PDF 内容。
- 载荷在读取后立即从地址栏移除，并在会话范围内保存。
- 日志默认不记录完整标题、作者、URL 或 DOI；调试模式也需脱敏。
- 扩展不向开发者或第三方服务器发送遥测。
- 任何页面结构异常均采用停止并提示，不采用模糊点击或自动提交。
- 插件仅辅助用户发布其本人需要的单篇文献求助，不提供批量、高频或代求助功能。

## 11. 错误处理

| 场景 | 行为 |
| --- | --- |
| 未选择文献 | 提示先选择一篇文献 |
| 选择多篇 | 提示首版仅支持单篇 |
| 选择附件或笔记 | 提示选择父级普通文献条目 |
| 标题缺失 | 阻止并提示补充标题 |
| 已有 PDF | 阻止并提示无需求助 |
| 浏览器扩展未安装 | 页面照常打开，提示可使用剪贴板元数据 |
| URL 载荷无效或过期 | 不填表，显示错误并清除载荷 |
| 未登录 | 暂存任务，等待用户登录 |
| DOI 查询失败 | 改用 Zotero 元数据手动填充 |
| 页面字段冲突 | 保留科研通结果并高亮差异 |
| 页面结构变化 | 停止自动化并提供复制元数据 |
| 打开浏览器失败 | Zotero 显示错误，剪贴板内容仍可用 |

## 12. 测试策略

### 12.1 共享逻辑单元测试

- DOI 标准化及非法 DOI 处理。
- 标题、作者、年份、期刊和 URL 规范化。
- 载荷编码、解码、版本检查、过期检查和长度限制。
- URL 协议白名单。
- 字段合并规则和差异检测。

### 12.2 Zotero 插件测试

- 未选、单选和多选行为。
- 非普通条目拒绝。
- PDF 内容类型、存储附件、链接附件和缺失附件判断。
- 元数据提取和剪贴板降级文本。
- 两个界面入口调用同一命令。
- 在 Zotero `9.0-beta.21+1a89239a1` 64 位中安装、启动、卸载和重启验证。

### 12.3 浏览器扩展测试

- Chrome 与 Edge 的 Manifest V3 加载。
- URL 片段读取后清除。
- 会话存储、登录恢复、超时与取消。
- DOI 查询成功、失败和超时。
- 空字段补齐及字段冲突提示。
- 无 DOI 填充。
- 页面选择器失效时停止操作。
- 确认任何路径都不会触发最终发布按钮。

### 12.4 端到端验收

使用至少以下样例在测试账号中验证：

1. 有 DOI、无 PDF、科研通查询成功。
2. 有 DOI、查询结果与 Zotero 标题不一致。
3. 无 DOI、有标题和官网 URL。
4. 无 DOI、无官网 URL。
5. 已有存储 PDF。
6. 已有链接 PDF。
7. 未登录后完成登录并恢复。
8. 默认浏览器未安装扩展的降级路径。

## 13. 验收标准

- `.xpi` 可在指定 Zotero 9 beta 版本安装并正常加载。
- Chrome 和 Edge 均可加载同一扩展代码。
- 单篇无 PDF 文献能从两个 Zotero 入口启动流程。
- 有 PDF、标题缺失、多选和错误条目类型会被正确阻止。
- 载荷不会发送给科研通服务器，且读取后从地址栏清除。
- 登录后能够恢复同一条文献任务。
- DOI 查询结果优先，Zotero 仅补空并提示差异。
- 无 DOI 时能够填写已有字段并提示补充官网链接。
- 页面变化时不会误点最终发布按钮。
- 所有自动化测试通过，并完成指定 Zotero、Chrome、Edge 实机验证。

## 14. 兼容性与维护

Zotero 9 当前为 beta 版本，内部接口和 UI 结构可能继续变化。插件清单应将最大兼容版本限制为已验证的小版本范围，并在升级 Zotero 后重新执行实机测试再放宽范围。

科研通页面同样可能调整。浏览器扩展将站点选择器集中管理，并为无法识别的页面提供明确降级路径。不得通过静默猜测维持表面上的“自动化成功”。
